Computational Thinking

Introduction

Computational thinking is the process of solving a problem in the most efficient manner possible. This proccess has many key elements, however it is commonly split into 4 parts:

  1. Decomposition
  2. Abstraction
  3. Pattern Recognition
  4. Algorithms

These elements (Commonly refered to as D.A.P.A.) allow us to solve problems in the easiest way avalable. A good example that includes all these particular items is the London Underground Map:

Decomposition

Decomposition is the process of breaking down a problem/task into smaller ones (or 'sub-problems'). This in turn will make the task at hand easier to solve, as each sub-problem can be worked on separatly, and in working in a team you can work on each task individual with promotes efficiency.

This applies to the previously mentioned example, The London Undergound Map, as each route/path a train takes, is separated with colours and names, instead of being one big system of tunnels. This helps making finding the right route more efficient as each path is seperated and is easier to find/access.

Abstraction

Abstarction is the procedure of removing unnecessary information, that is either irrelevant or redundant. Humans do this mostly everyday, without even noticing: e.g., humans, in a car, will put their foot on an accelerator to speed up or on the brake to slow down without knowing how the braking systems actually work. This helps solve a problem or perform a task, as it stops useless data bothering the person solving it.

For example, The London Underground is simpily a representation of itself; there are no trees, plants and otherwise irrelevant landmarks, that would make the map more ineffecient for the train passenger.

Pattern Recognition

Pattern Recognition is the action of identifying a problem and comparing it to a class of problems that have been previously solved. This makes sovling an issue much more efficient as you dont have to solve something that has already been solve (Doing this would be extremely redundant).

This may apply to The London Underground by the train user using the color code to see where he has to go next, instead of having to find his way around the complex tunnels of The Underground.

Algorithms

Algorithms are a step-by-step set of instructions to perform a task or solve a problem. Algorithms you can think as a sort of recipe, therefore humans use them all the time in cooking, blueprints for building and more.

As for The London Underground Map, algorithms are used to get to the place itself, board the train, get off at the right stop and etc. Algorithms are really the core of Computational Thinking.
